# embedded-self-check-playbook

임베디드(펌웨어) 개발자가 **로컬 LLM**으로 "스스로 점검하고 테스트하는" 개발 루프를 구축하기 위한 중립적 playbook.

핵심 주장은 하나다: **"자가치유(self-healing)"가 아니라 "자가점검(self-checking) 루프"** 다.
LLM은 후보를 *제안*하고, 결정론적 검증기(컴파일러·정적분석·유닛테스트·시뮬)가 *판정*하며, 사람이 되돌릴 수 없는 변경을 *승인*한다. **루프는 검증기만큼만 똑똑하다.**

> **성격.** 이 문서는 특정 제품·고객·사내 데이터를 포함하지 않는 중립 의사결정 지원 자료다. 도입 권유가 아니라 판단 근거다. 각 항목은 *Apply when / Skip when* 과 trade-off를 명시한다. 별점·arXiv ID·도구 상태는 조사 시점(2026-07) 기준이며 도입 전 재확인이 필요하다.

## 다루는 5가지 역량 (+ 원리)

| # | 역량 | 한 줄 | 검증기(load-bearing) |
|:-:|------|-------|----------------------|
| 0 | 자기점검 루프 원리 | LLM 제안 → 검증기 판정 → 사람 승인 | (전 항목 공통) |
| A | 코드 ↔ 사양 일치 점검 | 사양을 실행 가능한 테스트/어서션으로 | 유닛테스트, 트레이스 커버리지 |
| B | 신규 결함 → 현재 코드 자기점검 | "버그 하나, 뒤에 수백 개" 패턴 전파 | 정적 열거(semgrep/cppcheck) + 회귀 테스트 |
| C | 자기수정·자기테스트 루프 | 생성→실행→실패 피드백→재시도 | 컴파일러 + 테스트 러너 |
| D | 로컬 LLM + 검증기 스택 | 오프라인 소형모델을 신뢰 가능하게 | 제약생성·RAG·정적분석 |
| E | 규율·가드레일 | 자율성은 실수도 증폭한다 | 사람 게이트·mutation test |

## 한 줄 요약

로컬 소형 LLM은 "더 똑똑하게" 만들어서가 아니라 **출력을 제약하고, 입력을 그라운딩하고, 결정론적 검증기로 판정**해서 쓸모가 생긴다. 임베디드는 검증기(컴파일러·정적분석·유닛테스트·시뮬)가 풍부해 이 패턴에 특히 유리하다.
